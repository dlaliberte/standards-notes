Flexbox classes

General principals

This document defines classes that are used to create flexbox layouts based on the CSS Flexible Box Layout Module specification. The classes defined here are designed to be used in combination to create a wide variety of layouts with no additional CSS required in most cases.

The term "container" in the flexbox specification refers to an element that contains other elements, and the children elements of a container are referred to here as "items". An item can itself be a container of items.

Two container classes are defined here, "hbox" and "vbox". Variations of the container classes are also provided, which can be added as separate classes. For example, the hbox class can be combined with halign-items-center to create a horizontal container with items aligned to the center of the available width of the container.

Horizontal or Vertical Orientation

Container elements can be horizontal or vertical in orientation as determined by the class hbox or vbox, meaning the children of these containers will be arranged in rows or columns, respectively.

Note that the container in the following examples often have a larger than required fixed height to allow the items to expand vertically to fill the available space.

Justification versus Alignment

The term "justification" in the flexbox specification refers to shifting, stretching, or spacing items in the same direction as the item orientation. We can also shift, stretch, or space items in the "cross-axis" direction, which is the direction perpendicular to the item orientation. The flexbox specification refers to this as "alignment".

Therefore, when we shift, stretch, or space items in the same direction as the item orientation, we would have to use flexbox style properties that are confusingly named "justify-content", "align-content", or "flex-grow". And when we align items in the cross-axis direction, we would use flexbox style properties that are named "align-items" or "align-self" (with values of e.g. "flex-start").

Instead of using this flexbox terminology, every class that determines how we shift, stretch, or space items in a flexbox container starts with either halign- or valign-. And we determine whether "alignment" or "justification" apply based on whether the item orientation agrees with the alignment direction.

For example a container with class="hbox halign-items-center" will center items in the horizontal direction, and this is also true for a container with class="vbox halign-items-center". Similarly a container with class="hbox valign-items-center" will center items in the vertical direction, and this is also true for a container with class="vbox valign-items-center".

By default, items are shifted to the "start"-side of the container, which is the left side of an hbox and the top side of a vbox. Then we can add a halign-items-* class to align items in the horizontal direction, while the valign-items-* classes align items in the vertical direction.
