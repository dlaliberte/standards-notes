Flexbox classes

General principals

This document defines classes that are used to create flexbox layouts based on the CSS Flexible Box Layout Module specification. The classes defined here are designed to be used in combination to create a wide variety of layouts with no additional CSS required in most cases.

The term "container" in the flexbox specification refers to an element that contains other elements, and the children elements of a container are referred to here as "items". An item can itself be a container of items.

Two container classes are defined here, "hbox" and "vbox". Variations of the container classes are also provided, which can be added as separate classes. For example, the hbox class can be combined with halign-items-center to create a horizontal container with items aligned to the center of the available width of the container.

Horizontal or Vertical Orientation

Container elements can be horizontal or vertical in orientation as determined by the class hbox or vbox, meaning the children of these containers will be arranged in rows or columns, respectively.

Justification versus Alignment

The term "justification" in the flexbox specification refers to shifting, stretching, or spacing items in the same direction as the item orientation. We can also shift, stretch, or space items in the "cross-axis" direction, which is the direction perpendicular to the item orientation; the flexbox specification refers to this as "alignment".

Therefore, when we want to "justify" items, we would have to use flexbox style properties that are confusingly named "justify-content", "align-content", or "flex-grow". And when we want to "align" items in the cross-axis direction, we would use flexbox style properties that are named "align-items" or "align-self" (with values of e.g. "flex-start").

Instead of using this flexbox terminology, every class that determines how we shift, stretch, or space items in a flexbox container starts with either halign- or valign-. And we determine whether "alignment" or "justification" properties are used internally based on whether the item orientation agrees with the alignment direction.

For example a container with class="hbox halign-items-center" will center items in the horizontal direction, and this is also true for a container with class="vbox halign-items-center". Similarly a container with class="hbox valign-items-center" will center items in the vertical direction, and this is also true for a container with class="vbox valign-items-center".

By default, items are shifted to the "start"-side of the container, which is the left side of an hbox and the top side of a vbox. Then we can add one of the halign-items-* classes to align items in the horizontal direction, while the valign-items-* classes align items in the vertical direction.

Stretching

You'll notice above that, unless a cross-axis alignment is specified, the default behavior of flex boxes is to stretch the items in the cross-axis dimension. So by default, hbox items will stretch vertically and vbox items will stretch horizontally.

No matter what item-alignment is specified in the container, we can override the alignment for each item with a similarly-named class. For example, if we want to vertically align one item in an hbox, we can add the valign-start class to that item. Main-axis stretch alignment

The default behavior of flex boxes is to not stretch the items in the main-axis dimension. So by default, hbox items will shrink horizontally to the minimum required and vbox items will shrink vertically.

If at least one item is specified with a halign-stretch or valign-stretch class, all the other the items in the same axis will shrink to the minimum required size, unless they are also stretched.

If you want to stretch all items in an hbox proportionately, you can do so simply by adding the halign-items-stretch class to the container. Or if you want to stretch a single item in the main-axis dimension of an hbox, you can add the halign-stretch class to that item. Similarly, if you want to stretch all items in a vbox proportionately, you can do so simply by adding the valign-items-stretch class to the container. Or if you want to stretch a single item in the main-axis dimension of a vbox, you can add the valign-stretch class to that item. The next few examples show these classes in action.

Stretch ratios

Each item can be stretched a different amount by adding the halign-stretch-X or valign-stretch-X classes to each item. There are only three stretch ratios: 1, 2, and 3, and the default is 1. This amount of stretching is proportional to this ratio, and to the content. The next few examples show these classes in action.

Spacer

A set of spacer classes are provided as a convenience to add invisible spacing between items. You can insert one or more spacers in an hbox or vbox. (The cross-axis spacers would only apply when wrapping, but then we could not indicate which spacer would go where. So we don't need the halign- or valign- prefix.)

By adding spacers, you can reproduce the equal space justification classes, but you also have much more flexibility to justify and align in many more ways. There are actually four spacer classes with different expansion weights, similar to the halign-stretch-* and valign-stretch-* classes: spacer, spacer-1, spacer-2, and spacer-3.

Reversing and ordering

All the items in a container can be reversed by adding the 'reverse' class to the container. The order of the items is the reverse of the original order, and the main-axis alignment is also reversed, from start to end or vice versa.
